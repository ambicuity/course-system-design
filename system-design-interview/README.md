# System Design Interview — Course Notes

Markdown notes for a *System Design Interview* course (chapters 02–30).

## About this course

The original chapter files were plain markdown notes — useful but shallow. The
notes deepen every chapter to **interview-grade** depth by appending five
enrichment sections to each one:

1. **Back-of-the-Envelope Math** — concrete numbers using powers of 2, with the
   arithmetic shown step by step.
2. **ASCII Architecture Diagrams** — sequence diagrams, system block diagrams,
   and data-flow diagrams rendered in fenced code blocks.
3. **Trade-off Tables** — 3+ options compared across latency, complexity, cost,
   consistency, and operational burden.
4. **Real-World Case Studies** — what Twitter, Discord, Stripe, Dropbox, Uber,
   Cloudflare, LinkedIn, Netflix, Google, Amazon, etc. actually did, and why.
5. **Common Pitfalls & Failure Modes** — the anti-patterns that cause outages
   in production and tank interviews.
6. **Interview Q&A** — sharp follow-up questions a senior interviewer actually
   asks, with crisp answer sketches.
7. **Glossary** — 8+ technical terms with common misconceptions vs. precise
   definitions.

Target length per chapter: **600–1200 lines**.
